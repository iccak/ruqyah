# الرقية الشرعية — مشغل رقية مخصص (بدون إنترنت)

تطبيق فلاتر (Android) لكبار السن، بواجهة عربية بالكامل (RTL)، لإنشاء
قوائم تشغيل مخصصة من مقاطع صوتية (سور) مستوردة من الجهاز، مع تحديد
عدد تكرار كل مقطع، وتشغيلها تسلسليًا في الخلفية حتى عند إغلاق الشاشة.

## المزايا الرئيسية
- عمل كامل بدون إنترنت (الملفات تُنسخ وتُخزَّن داخل التطبيق).
- إنشاء عدة "رقيات" مخصصة، كل واحدة بقائمة سور وتكرارات مختلفة.
- تشغيل تسلسلي: يكمل تكرار المقطع الأول بالكامل قبل الانتقال للتالي.
- استمرار التشغيل في الخلفية مع إشعار دائم وتحكم من شاشة القفل.
- واجهة عالية التباين وخطوط كبيرة مناسبة لكبار السن.

## هيكل المشروع
```
lib/
  models/            نماذج البيانات (AudioItem, Ruqyah, RuqyahTrack) + محولات Hive
  services/          التخزين المحلي (Hive) + معالج الصوت في الخلفية (audio_service)
  screens/           الشاشة الرئيسية، مكتبة السور، محرر الرقية، شاشة التشغيل
  theme.dart         ثيم موحّد عالي التباين
  main.dart          نقطة انطلاق التطبيق (RTL + تهيئة الخدمات)
android/             إعدادات أندرويد (AndroidManifest, Gradle, أيقونات)
.github/workflows/   بناء APK تلقائي عبر GitHub Actions
```

## البناء عبر GitHub Actions (بدون أي جهاز كمبيوتر)
1. أنشئ مستودعًا جديدًا على GitHub وارفع محتوى هذا المجلد إليه.
2. توجّه إلى تبويب **Actions** في المستودع؛ سيبدأ سير العمل
   `Build APK` تلقائيًا مع كل رفع (push) إلى الفرع `main`،
   أو شغّله يدويًا عبر زر **Run workflow**.
3. بعد انتهاء التنفيذ بنجاح، افتح نتيجة التشغيل وحمّل ملف
   **ruqyah-app-release-apk** من قسم **Artifacts** — يحتوي على
   `app-release.apk` الجاهز للتثبيت.

> لإصدار رسمي (Release) على صفحة GitHub Releases، أضف Git tag يبدأ
> بحرف (مثل `v1.0.0`) وارفعه؛ سيقوم سير العمل بإرفاق الـ APK تلقائيًا
> بالإصدار.

## البناء المحلي (اختياري)
```bash
flutter pub get
flutter run          # للتجربة على جهاز/محاكي متصل
flutter build apk --release
```
الملف الناتج يكون في: `build/app/outputs/flutter-apk/app-release.apk`

## ملاحظات مهمة
- **التوقيع (Signing):** يستخدم إعداد `release` الحالي توقيع التصحيح
  الافتراضي (`debug` signing) لتسهيل البناء التلقائي المباشر. لنشر
  التطبيق على متجر Google Play، يجب إنشاء مفتاح توقيع خاص بك وتحديث
  `android/app/build.gradle` ليستخدمه (راجع توثيق Flutter الرسمي حول
  "Signing the app").
- **أيقونة التطبيق:** الأيقونات الحالية في `android/app/src/main/res/mipmap-*`
  هي أيقونات مؤقتة (placeholder) بلون التطبيق الأساسي؛ استبدلها بشعارك
  الخاص عند الحاجة (يمكن استخدام حزمة `flutter_launcher_icons`).
- **الخط العربي:** الثيم يشير إلى خط باسم `Cairo`؛ في حال عدم إضافة
  ملفات الخط ضمن `pubspec.yaml`/`assets`، سيسقط النظام تلقائيًا لاستخدام
  خط النظام الافتراضي دون أي كسر بالتطبيق.
- **محولات Hive (`*.g.dart`):** مرفقة جاهزة يدويًا في هذا المستودع، لذا
  لا حاجة لتشغيل `build_runner` للبناء الأساسي؛ خطوة توليدها في سير
  عمل GitHub Actions اختيارية (`continue-on-error`) لإعادة توليدها تلقائيًا
  إذا عدّلت الموديلات مستقبلًا.
